# Project: BMAD Task Tracker CLI
## Role: Senior Node.js Engineer
## Tech Stack: Node.js (ESM), FileSystem (fs/promises)

## Rules & Constraints:
- Use Clean Code principles.
- Single Responsibility: Separate data logic from CLI output.
- Data Persistence: Store tasks in `tasks.json`.
- Error Handling: Handle missing files or invalid JSON gracefully.

## Features:
- `add [text]`: Adds a task.
- `list`: Shows all tasks with IDs.
- `remove [id]`: Deletes a task by ID.
